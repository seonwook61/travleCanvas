# trip-canvas PRD

## 제품 한 줄

일본 자유여행을 준비하는 한국 사용자가 장소를 저장하고, 여행 단위로 묶고, 날짜별 itinerary로 정리한 뒤 읽기 전용 링크로 공유할 수 있는 웹앱.

## 문제 정의

일본 자유여행 준비 과정은 보통 검색, 장소 저장, 지도 확인, 일정 조합, 링크 공유가 여러 앱과 메모로 분산된다. 사용자는 장소를 저장하는 단계와 실제 여행 계획으로 발전시키는 단계 사이에서 맥락을 자주 잃는다.

## 타깃 사용자

- 일본 자유여행을 준비 중인 20~30대 한국 사용자
- 여행 후보지를 먼저 많이 모으고, 나중에 일정으로 정리하는 사용자
- 저장한 장소와 여행 동선을 친구나 동행자에게 깔끔하게 공유하고 싶은 사용자

## 핵심 가치 제안

- 장소를 먼저 저장하고 나중에 trip으로 승격시키는 자연스러운 흐름
- 일본 전역 검색이 가능하지만 주요 도시 우선 UX로 시작하는 낮은 진입 장벽
- 저장 중심 라이브러리와 가벼운 day-by-day itinerary의 결합
- 읽기 전용 공유 링크로 여행 결과물을 깔끔하게 전달

## 제품 원칙

- 메인 경험은 `저장`이다.
- itinerary는 저장한 장소를 정리하는 두 번째 레이어다.
- 로그인 전 탐색은 허용하고 저장 시 로그인한다.
- 모든 trip은 기본 비공개이며 공유는 명시적 액션으로만 생성한다.
- UI 기본 언어는 한국어다.
- MVP 로그인 제공자는 `Google OAuth only`로 고정한다.

## MVP 범위

- 지도와 검색으로 일본 장소 탐색
- 비로그인 탐색, 저장 시 로그인
- 계정 전체 장소 라이브러리
- `wishlist / visited / favorite` 3단계 저장 상태
- 저장한 장소를 기반으로 trip 생성
- 실제 날짜 + Day 병행 itinerary
- 짧은 텍스트 메모
- 읽기 전용 `여행 상세 전체 공유 링크`

## Non-goals

- 결제
- 실시간 예약
- 협업 편집
- 시간 단위 세부 일정 편집
- 자동 최적 경로 추천
- 이미지 업로드 메모
- 기본 공개 여행
- 일본 외 국가 확장

## 핵심 사용자 흐름

1. 비로그인 사용자가 일본 장소를 검색하고 지도에서 탐색한다.
2. 저장 액션을 시도하면 로그인한다.
3. 로그인 후 장소를 `wishlist`, `visited`, `favorite` 중 하나로 저장한다.
4. 저장한 장소 중 일부를 선택해 trip을 생성한다.
5. trip에 시작일/종료일을 넣고 Day별로 장소를 배치한다.
6. 완성된 trip을 읽기 전용 링크로 공유한다.

## 주요 화면

- Map Home
- Saved Places Library
- Trip Create / Organize
- Trip Detail
- Shared Trip View
- Auth Entry

## 성공 지표

- 장소 저장 전환율
- 저장한 장소에서 trip 생성으로 이어지는 비율
- trip 생성 후 공유 링크 생성 비율
- 재방문 시 저장 라이브러리 재활용 비율

## 개인정보 / 공유 원칙

- trip은 기본 비공개
- 사용자가 공유를 명시적으로 생성할 때만 링크 발급
- 공유 링크는 읽기 전용
- 공유 링크는 `Trip Detail`에서 명시적으로 생성하고 복사한다.
- 메모는 짧은 텍스트만 허용

## Phase 2 확장 아이디어

- 모바일 전용 레이아웃 최적화
- 주요 도시 큐레이션 강화
- 협업 편집
- 이미지 첨부 메모
- 추천 경로 보조 기능

## 구현 확정 사항

- 인증 제공자는 `Google OAuth only`
- 공유 링크는 만료 없이 생성되며, 현재는 수동 재생성 방식으로만 운영
- 공개 공유 페이지는 `핀보드 컬렉션 + 루트 보드` 혼합 구조를 사용
