# Documentation Guide

`trip-canvas` keeps product-definition artifacts in `docs/`.

## Canonical Documents

- `prd/PRD.md`: product direction, users, MVP, and success metrics
- `specs/product-spec.md`: detailed requirements and acceptance criteria
- `specs/ERD.md`: entity model and ownership
- `api/API_SPEC.md`: initial API contract draft
- `screens/SCREEN_DEFINITION.md`: screen purpose, states, and actions
- `reports/DECISION_LOG.md`: major decisions locked during brainstorming

## Superpowers Integration

The repo uses product-first docs instead of the default `docs/superpowers/specs/` pattern.

- brainstorming outputs should be reflected in the canonical docs above
- implementation plans should be written under `docs/superpowers/plans/`
